## Git的实用笔记

`git add .` 将当前状态读入缓存  
`git commit -m "2026-1-3 1st commit"` 对缓存标注状态  
`git pull` 同步远程仓库  
`git push -u origin master` 推送更新. 注意要提前设置好远程仓库为`origin`, 以及你要推送的branch为`master`  
`git remote set-url origin https://github.com/RexWang1981/website.git`设置远程仓库为`origin`  
`git remote -v`查看看远程仓库地址  
`git fetch`拉取远程仓库变更给, 但不`merge`  
`git pull` 拉取远程仓库,并`merge`. `pull`=`fetch`+`merge`  
有时`git pull`或`git push -u orign master`失败是因为网络连接问题.更换wifi就能解决问题  
`git rebase`可以解决很多`merge`遇到的问题  
`git rebase`是用来合并分支记录的  
`git merge home`表示从branch`honm`中合并内容到`master`,假定当前分支为`master`  
`git branch` 查看当前branch情况
`git switch home` 切换到branch`home`
